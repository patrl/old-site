Resources

Linguistics

  • Despite its famously unreliable servers, LingBuzz is still the one-stop shop for open access papers in generative linguistics.
  • Semantics Archive is an open access archive for semantics papers.
  • UCL linguistics publish an annual volume of working papers, which can be found here.
  • I've just started using pst-jtree for drawing syntax trees in LaTeX. The syntax is a little opaque, but it's incredibly flexible.
  • Norbert Hornstein's blog Faculty of Language is essential reading for generative linguists.
  • Gillian Ramchand's blog.

Other

  • Try Haskell in your browser; If you're a semanticist with some interest in learning to code (like me), then this is worth checking out.
  • Along similar lines, Natural Language Processing with Python is a free textbook which provides an excellent way for linguists to get into coding with python.

My setup

I'm running Tumbleweed, OpenSUSE's fantastic rolling linux distro. I spend the vast majority of my time in emacs. Here's a nice quote from Neal Stephenson (1998; In the Beginning was the Command Line):

...the engineer-hours that, in the case of Microsoft Word, were devoted to features like mail merge, and the ability to embed feature-length motion pictures in corporate memoranda, were, in the case of emacs, focused with maniacal intensity on the deceptively simple-seeming problem of editing text. If you are a professional writer – i.e., if someone else is getting paid to worry about how your words are formatted and printed – emacs outshines all other editing software in approximately the same way that the noonday sun does the stars. It is not just bigger and brighter; it simply makes everything else vanish.
